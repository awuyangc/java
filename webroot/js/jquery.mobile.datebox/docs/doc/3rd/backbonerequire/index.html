File Definition and Build

paths:: { "jqmdatebox": "external/jqmdatebox/jqm-datebox", }

To build a combined file, take a look at the 'custombuild.py' script in the build directory - alternativly, grab the compressed or uncompressed files for the mode(s) you want, and append them to the bottom of the .core.js file - order shouldn't matter, so long as core appears first. This file can be minimized

Shim Section

"jqmdatebox": {
"deps": ["jquery", "jquerymobile"]
}

Basically, Datebox has to load after jquerymobile. There may be a better way of doing this, but I don't know it

Defaults and Enhancment

Just this wasn't enough, the enhancment code was loading too late - so, add this bit to the require:

require( ["jqmdatebox"] , function() {
jQuery.extend(jQuery.mobile.datebox.prototype.options, {
'useNewStyle': true,
'useFocus': true,
'useInlineBlind': true,
});
$( ":jqmData(role='datebox')" ).each(function() {
var defed = typeof ($(this).data('mobile-datebox'));
if ( defed === "undefined" ) {
$(this).datebox();
}
});	
});

The first bit is the defaults for the datebox. Second bit is a copy of the default auto-enhance code.

If you want to progmatically call the enhance, you can omit both these bits, so long as you still require jqmdatebox - otherwise, no script to do the enhancment
